# 🏠 Willhaben Wohnungs-Watcher → Telegram

Автоматический мониторинг квартир на Willhaben (Вена). Раз в час проверяет новые
объявления, фильтрует по твоим критериям, помечает приоритетные и присылает алерт
в Telegram. Работает в облаке (GitHub Actions) — компьютер держать включённым не нужно,
всё бесплатно.

---

## ⚡ Настройка за ~10 минут (3 шага)

### Шаг 1. Создай Telegram-бота и узнай chat_id

1. Открой в Telegram **@BotFather** → отправь `/newbot` → задай имя и username.
2. BotFather пришлёт **токен** вида `7712345678:AAExxxxxxxxxxxxxxxxxxxxxxxxxxxxx`. Сохрани.
3. Напиши своему новому боту любое сообщение (например `старт`) — это обязательно.
4. Узнай свой **chat_id**: открой в браузере (подставь токен):
   `https://api.telegram.org/bot<ТОКЕН>/getUpdates`
   В ответе найди `"chat":{"id":123456789` — это число и есть твой `chat_id`.

### Шаг 2. Скопируй свой URL поиска с Willhaben

1. Зайди на **willhaben.at** → Immobilien → **Mietwohnungen** → регион **Wien**.
2. Выстави фильтры, которые доступны в интерфейсе:
   - **Preis bis: 1250**
   - **Zimmer: ab 3**
   - Сортировка: **Neueste zuerst** (важно!)
3. Скопируй URL из адресной строки браузера — это твой `SEARCH_URL`.

> Остальные фильтры (кухня, ohne Ablöse, не Gemeinde/WG, дата заезда, приоритеты)
> уже зашиты в скрипт — их в интерфейсе Willhaben нет.

### Шаг 3. Залей на GitHub и включи

1. Создай бесплатный аккаунт на **github.com** (если нет).
2. Создай **новый репозиторий** (New repository) → назови, например, `willhaben-watcher` →
   тип **Private** → Create.
3. Загрузь в него все файлы из этой папки (кнопка **Add file → Upload files**, перетащи всё,
   включая папку `.github`). Commit.
4. Открой **Settings → Secrets and variables → Actions → New repository secret** и добавь:
   | Name | Value |
   |------|-------|
   | `TELEGRAM_TOKEN` | токен из шага 1 |
   | `TELEGRAM_CHAT_ID` | chat_id из шага 1 |
   | `WILLHABEN_SEARCH_URL` | URL из шага 2 |
5. Открой вкладку **Actions** → согласись включить workflows → выбери
   **willhaben-watcher** → **Run workflow** (ручной запуск для проверки).

Через минуту в Telegram придёт сообщение «✅ Willhaben-watcher запущен…» + до 5 самых
релевантных квартир. Дальше бот сам будет присылать **только новые** объявления каждый час.

## 🎯 Что фильтрует (уже настроено)

**Жёсткие условия** (иначе объявление не присылается):
- цена ≤ **1250 €** (включая Betriebskosten)
- комнат ≥ **3**
- встроенная **кухня** есть
- **ohne Ablöse**
- не **Gemeindewohnung**, не **WG**
- заезд **не позже сентября** (октябрь+ отсекается)

**Приоритетные уведомления** 🔥 (помечаются отдельно, вверху сообщения):
- включено **паркоместо** / есть **бассейн** / **спортзал**
- при цене ≤ 1200 € включены **электричество + вода**
- **3 комнаты дешевле 900 €**

**Приятные бонусы** (просто помечаются в сообщении): 🌿 балкон · 🅿️ паркоместо · 📦 Kellerabteil.

## 🔧 Как менять критерии

Всё вверху файла `willhaben_watcher.py` в блоке `# НАСТРОЙКИ`. Отредактируй число/`True`/`False`,
закоммить — при следующем прогоне применится. Примеры:
- поднять бюджет: `MAX_TOTAL_PRICE = 1300`
- убрать фильтр по кухне: `REQUIRE_KITCHEN = False`
- смягчить дату заезда: `HARD_FILTER_MOVE_IN = False`

**Проверять раз в 30 минут:** в `.github/workflows/willhaben.yml` замени
`cron: "0 * * * *"` на `cron: "0,30 * * * *"`.

## ℹ️ Важные нюансы

- **Первый запуск** ничего не «спамит»: помечает текущие квартиры как виденные и шлёт
  короткую сводку + 5 примеров. Реальные алерты идут со второго прогона.
- GitHub отключает расписание, если в репозитории **60 дней нет активности** (придёт письмо,
  включается одной кнопкой). Любой коммит сбрасывает счётчик; бот сам коммитит `seen.json`,
  так что на практике это не мешает.
- Если после ручного запуска в логах Actions видишь `⚠️ __NEXT_DATA__ не найден` —
  Willhaben сменил разметку или временно заблокировал IP. Пришли мне лог из вкладки
  Actions, подправлю парсер.
